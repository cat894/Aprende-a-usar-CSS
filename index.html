Intermedio Práctico Sin frameworks

1) Especificidad: ¿qué regla gana?

La “fuerza” de una regla depende del selector: id > clase > tipo. Y lo último escrito también puede ganar si la especificidad es igual.

/* tipo < clase < id */
div.caja { background: #fee2e2; }
.especial { background: #fde68a; }
#muy-importante { background: #bbf7d0; }

2) Modelo de caja (Box Model)

Todo elemento tiene contenido, padding, border y margin. El tamaño total incluye esos “anillos”.

.box {
  background: #e0e7ff;   /* color de fondo */
  border: 4px solid #6366f1; /* borde */
  padding: 16px;          /* relleno interno */
  margin: 12px 0;         /* separación con otros elementos */
}
Soy una caja con padding, borde y margen. Observa los espacios.

3) Flexbox básico (distribución en fila)

Flexbox te ayuda a alinear y distribuir elementos fácilmente.

.contenedor { display: flex; gap: 10px; }
.item { flex: 1; }

4) Diseño responsive con @media

Usa media queries para cambiar el layout según el ancho de pantalla (móvil → desktop).

.cards { display: grid; grid-template-columns: 1fr; }
@media (min-width: 640px) {
  .cards { grid-template-columns: repeat(3, 1fr); }
}

5) Variables CSS + tema sin JavaScript

Las variables (custom properties) te permiten cambiar colores y estilos en bloque.

:root { --bg: #fff; --fg: #111; }
.caja { background: var(--bg); color: var(--fg); }
